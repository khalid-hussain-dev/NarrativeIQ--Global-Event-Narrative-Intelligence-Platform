# NarrativeIQ Exhibition Demo Flow

This is the recommended judge-facing flow for the Data Warehousing and Big Data lab exhibition.

## 1. Start The Demo

From the project root:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start_demo.ps1
```

For the safest exhibition setup with preloaded saved briefs:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start_demo.ps1 -SeedDemoBriefs
```

Useful options:

- `-BuildFrontend`: rebuild the production frontend before launch.
- `-RefreshData`: regenerate the 50,000-record warehouse/mart.
- `-SkipData`: skip dataset generation checks.
- `-SkipReport`: skip HTML report regeneration.

The script starts:

- Dashboard: `http://localhost:3000`
- API: `http://127.0.0.1:8000`
- API docs: `http://127.0.0.1:8000/docs`

Logs are written to:

```text
logs/backend-demo.out.log
logs/backend-demo.err.log
logs/frontend-demo.out.log
logs/frontend-demo.err.log
logs/admin_audit.jsonl
```

## 2. Opening Story

Say:

> NarrativeIQ is a warehouse-backed narrative intelligence platform. It takes event, topic, entity, sentiment, and source signals, transforms them through an ETL pipeline, stores them in a star-schema warehouse, and exposes dashboard marts for analysis.

Then show the landing/overview screen.

Point out:

- Full logo on landing.
- Icon-only dashboard signal.
- FastAPI mart status strip.
- Warehouse files count.
- 50,000 generated content records.
- Warehouse quality score.

## 3. Topic IQ Live Search

Go to **Topic IQ**.

Search one strong topic:

```text
OpenAI
```

Then search one surprising topic:

```text
Beyblade
```

Explain:

- The brief uses public reference/news metadata.
- Confidence is calculated from signal strength, source diversity, relevance, reference depth, and recency.
- Evidence metrics separate total signals, reference signals, news signals, and source clusters.

Click **Save Brief**.

## 4. Live Warehouse History

Scroll to **Live Warehouse History** under Topic IQ.

Show:

- PostgreSQL saved snapshots.
- Snapshot key.
- Topic name.
- Confidence.
- Total signals.
- Source rows.

Click **Load Brief** on a saved row.

Say:

> This proves the live search is not only a UI result. It is persisted into PostgreSQL through `dim_live_topic`, `fact_live_topic_snapshot`, and `live_topic_sources`.

## 5. Dashboard Analytics

Show these sections:

- **Events**: event dimension and event-level metrics.
- **Sentiment**: sentiment fact aggregation.
- **Replay**: narrative lifecycle over time.
- **Compare**: side-by-side narrative strength, growth, influence, sentiment, and monthly trajectory.
- **Graph**: entity/narrative relationships with zoom, pan, fit view, focus, and selected-node expansion.
- **Predictions**: forecast-style analytics.
- **Data Quality**: ETL quality profiling.

Key line:

> The dashboard is powered by the mart generated from the warehouse layer, not hard-coded cards.

## 6. Reports

Go to **Reports**.

Click:

1. **Export PDF**
2. **Open HTML**
3. **Export CSV**

Explain:

- HTML and PDF use the full NarrativeIQ report logo.
- PDF export is generated from the branded HTML report through headless Chrome/Edge.
- CSV export provides a quick summary artifact.

## 7. Administration

Go to **Administration**.

Show:

- Mart record count.
- Warehouse quality.
- PostgreSQL configured status.
- ETL run controls.
- Loader dry-run/load controls.
- System Audit Log for ETL, warehouse, report, and live-topic save actions.

Avoid running heavy ETL during the live judging unless asked.

## 8. Viva Cheat Sheet

If asked where data comes from:

- Main dashboard mart: deterministic 50,000-record exhibition dataset generated by `etl/pipeline.py`.
- Topic IQ: live public-source metadata from Wikipedia, GDELT, and Google News RSS.
- Saved live briefs: persisted in PostgreSQL live-topic tables.

If asked whether the project uses a data warehouse:

- Yes. It has dimension tables, fact tables, materialized views, warehouse CSV outputs, PostgreSQL loading, and API-ready marts.

If asked what Big Data concept is shown:

- High-volume generated content, source/event/topic/entity dimensions, time-series facts, sentiment and trend measures, quality profiling, graph relationships, and mart-based analytics.

If asked what remains for a production system:

- Scheduled ingestion, authentication, automated tests, richer source connectors, and deployment hardening.
