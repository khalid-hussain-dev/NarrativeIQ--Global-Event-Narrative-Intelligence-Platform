# NarrativeIQ

NarrativeIQ is a Global Event and Narrative Intelligence Platform for a Data Warehousing and Big Data lab project. It turns fragmented public discourse into historical intelligence by modeling events, narratives, entities, sentiment, and predictions through a warehouse-first architecture.

## Exhibition Positioning

**Tagline:** Transforming digital conversations into narrative intelligence.

The project is built to demonstrate:

- Star-schema data warehouse with dimension and fact tables.
- ETL generation, profiling, cleaning, enrichment, and loading.
- Narrative lifecycle analysis across time.
- Sentiment evolution, entity intelligence, and relationship graphs.
- Prediction center and narrative replay mode for the exhibition demo.

## Exhibition Quick Start

Run the complete local exhibition demo from the project root:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start_demo.ps1
```

For a judge-safe setup with sample saved live-topic briefs preloaded:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start_demo.ps1 -SeedDemoBriefs
```

Then open [http://localhost:3000](http://localhost:3000).

The startup script health-checks FastAPI and Next.js, writes logs into `logs/`, and prints the dashboard/API/report paths when ready. Use `-BuildFrontend` to rebuild the production frontend, or `-RefreshData` to regenerate the 50,000-record warehouse mart.

Recommended exhibition walkthrough:

- `docs/EXHIBITION_DEMO_FLOW.md`

## Developer Quick Start

Generate assets and data:

```powershell
python scripts/create_logo_assets.py
python etl/pipeline.py --records 50000
```

Run the backend:

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Run the frontend:

```powershell
cd frontend
npm.cmd install
npm.cmd run dev
```

Then open [http://localhost:3000](http://localhost:3000).

## Warehouse Loading

The ETL pipeline writes warehouse CSV outputs first. To load them into PostgreSQL, configure a connection string and run the loader:

```powershell
$env:NARRATIVEIQ_DATABASE_URL="postgresql://postgres:password@localhost:5432/narrativeiq"
python etl/load_postgres.py --dry-run
python etl/load_postgres.py
```

Recommended local configuration:

```powershell
Copy-Item .env.example .env
notepad .env
createdb -U postgres narrativeiq
powershell -ExecutionPolicy Bypass -File scripts/check_postgres.ps1
powershell -ExecutionPolicy Bypass -File scripts/check_postgres.ps1 -LoadWarehouse
```

The `createdb` command only needs to be run once, and PostgreSQL must already be installed with `psql`/`createdb` on PATH. The `.env` file is intentionally ignored by Git. Once `NARRATIVEIQ_DATABASE_URL` is set, the FastAPI admin status changes from PostgreSQL pending to configured when the backend is started through `scripts/start_demo.ps1`.

Example `.env` value:

```env
NARRATIVEIQ_DATABASE_URL=postgresql://postgres:password@localhost:5432/narrativeiq
```

The FastAPI admin endpoints can also run ETL and trigger the loader:

- `GET /admin/status`
- `POST /admin/etl/run?records=50000`
- `POST /admin/warehouse/load?dry_run=true`
- `POST /admin/warehouse/load?dry_run=false`
